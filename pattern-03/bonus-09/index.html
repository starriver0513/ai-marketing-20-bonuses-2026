BONUS 09 / PRODUCT CREATION SKILL

おさる式商品作成Skill高単価講座設計対応版

おさる式の商品作成メソッドに沿って、商品コンセプト、講座カリキュラム、サポート設計、販売導線まで整理できる商品作成スキルです。最初から商品アイデアを完璧にまとめておく必要はありません。作りたい商品の概要を伝えると、AIが必要な項目を順番に確認しながら商品設計を進めます。

01

商品作成スキルを受け取る

商品作成スキルのフォルダ一式を受け取れます。受け取ったフォルダをCodexまたはClaude Codeが読める場所に配置すると、商品作成のヒアリングを進められます。

商品作成スキルを受け取る 商品作成スキルのフォルダ一式を開く
02

Codex / Claude Codeで使える状態にする

商品作成スキルは、フォルダ一式を受け取ってから、CodexまたはClaude Codeが読み取れる場所に配置して使います。参加者は難しい設定を覚える必要はありません。デスクトップアプリで作業フォルダを開き、商品作成スキルを使いたいと伝えれば、AIが必要項目を順番に確認します。

01 商品作成スキルを受け取る

ボタンから商品作成スキルのフォルダ一式を受け取り、PC内の分かりやすい場所に保存します。

02 商品作成用の作業フォルダを作る

商品案、講座メモ、過去コンテンツ、顧客の声など、商品作成に使いたいファイルをまとめる作業フォルダを用意します。

03 Codexで使う場合

作業フォルダ内に .agents/skills フォルダを作り、その中へ商品作成スキルのフォルダを入れます。その作業フォルダをCodexアプリのプロジェクトとして開きます。

04 Claude Codeで使う場合

作業フォルダ内に .claude/skills フォルダを作り、その中へ商品作成スキルのフォルダを入れます。複数の作業フォルダで共通利用したい場合は、個人用の ~/.claude/skills に置く方法もあります。

05 自然に作成を依頼する

CodexまたはClaude Codeに「商品作成を始めたいです」「このスキルを使って講座設計を進めたいです」のように伝えます。すると、AIが商品概要やターゲットなどを順番に確認します。

06 情報源になるファイルを一緒に渡す

すでに商品案、講義メモ、相談内容、既存コンテンツ、顧客の声がある場合は、ファイルごと作業フォルダに入れて分析させると、ヒアリングと商品設計が具体的になります。

03

この商品作成スキルでできること

このスキルでは、作りたい商品の概要を伝えると、AIが必要な項目を順番に確認します。回答を進めることで、商品アイデアを販売できる高単価講座・コンテンツ販売商品へ具体化できます。

CONCEPT 商品コンセプト

誰に、どんな悩みを、どんな独自の切り口で解決するのかを整理します。

PRICE 価格とプラン

提供価値、サポート範囲、顧客の変化量をもとに松竹梅のプランを整理します。

COURSE 講座カリキュラム

受講生が理想状態へ進むための章立て、ワーク、課題、補足教材を設計します。

SUPPORT サポート設計

個別面談、チャット、添削、グルコン、コミュニティなど提供範囲を決めます。

OFFER 販売導線

無料相談、モニター販売、通常販売、講座化、スクール化までの流れを整理します。

IMPROVE 改善ログ

顧客の質問、つまずき、成果、反応を次の商品改善へ戻します。

04

商品作成はヒアリングから始まる

最初から市場情報や商品設計をすべて用意する必要はありません。作りたい商品の概要を伝えると、ターゲット、悩み、理想状態、提供できる内容などを順番に確認されます。その回答をもとに、商品作成が始まります。

01 作りたい商品の概要を伝える

まずは、どんな商品を作りたいのかをざっくり伝えます。ジャンル、対象者、解決したい悩みが曖昧でも問題ありません。

02 AIからヒアリングを受ける

AIが、ターゲット、悩み、理想状態、提供できる価値、既存の経験、サポート内容などを順番に質問します。

03 分かる範囲で回答する

すべてを完璧に答える必要はありません。分からない項目は、AIに整理してもらいながら進めます。

04 必要に応じてAIに調査させる

市場や競合の確認が必要な場合は、AIに調査させます。最初からリサーチ資料を用意して渡す前提ではありません。

05 商品設計に進む

ヒアリング内容をもとに、コンセプト、カリキュラム、サポート、特典、販売導線の作成へ進みます。

05

ヒアリングされる主な項目

商品作成スキルでは、AIが下記のような項目を順番に確認します。質問に直接答えても進められますが、情報源になるファイルがある場合は、そのまま渡して分析させるのがおすすめです。過去の講義メモ、商品案、顧客の声、既存コンテンツ、相談内容などを読み込ませることで、より具体的な商品設計にできます。

PRODUCT 作りたい商品の概要

何を教える商品にしたいのか、どんな変化を支援したいのかを確認します。

TARGET ターゲット

誰に向けた商品なのか、その人が今どんな悩みを抱えているのかを確認します。

GOAL 理想状態

商品を受けた後に、顧客がどんな状態になれば成功なのかを明確にします。

ASSET 原資産

ユーザーの経験、実績、既存コンテンツ、支援できることを整理します。

OFFER 提供内容

動画講義、ワーク、添削、面談、チャット、コミュニティ、特典など提供できる内容を確認します。

CHECK 不足情報

商品作成に足りない情報があれば、追加質問やAIリサーチで補います。

06

商品作成スキルの使い方

スキルフォルダを配置できたら、CodexまたはClaude Codeのデスクトップアプリで作業フォルダを開き、作りたい商品について自然に伝えます。AIからの質問に答えながら、商品コンセプト、カリキュラム、提供内容、販売導線を具体化していきます。

01 作業フォルダを開く

CodexまたはClaude Codeで、商品作成に使うファイルをまとめた作業フォルダを開きます。

02 商品作成を始める

「商品作成を始めたいです」「講座設計を進めたいです」のように、自然な言葉で依頼します。

03 AIのヒアリングに答える

ターゲット、悩み、理想状態、提供できる内容、既存コンテンツなどを分かる範囲で答えます。

04 情報源ファイルを分析させる

関連するメモや既存資料がある場合は、ファイルごと渡して分析させると、商品設計の精度が上がります。

05 出力を確認する

コンセプト、カリキュラム、講座内容、サポート内容がイメージと合っているか確認します。

06 フィードバックして整える

違和感がある箇所は、そのままAIに伝えます。納得できるまで修正し、販売できる商品案に近づけます。

07

使うタイミング

商品作成スキルは、商品アイデアを形にしたい時、既存の個別支援を講座化したい時、コンテンツ販売商品の中身を作り込みたい時に使います。作りたい内容を自然に伝えるとAIが必要項目をヒアリングし、おさる式の商品作成メソッドで商品コンセプト、カリキュラム、提供内容、販売導線を具体化していきます。

  • 新しい高単価講座やコンテンツ販売商品を作りたい時。
  • 個別支援やコンサルを、講座・スクール型の商品へ変えたい時。
  • 既存コンテンツを整理して、体系的なカリキュラムにしたい時。
  • 市場や競合を見たうえで、商品のコンセプトを見直したい時。
  • 無料相談やモニター販売で得た質問・反応を、商品改善に反映したい時。
08

導入時に確認する公式ページ

CodexとClaude Codeのスキル配置は、公式ドキュメント上でも決められています。導入で迷った場合は、下記も確認してください。

Codex Skills公式ドキュメント Codexが読み取る .agents/skills の配置ルールを確認できます。 Claude Code Skills公式ドキュメント Claude Codeが読み取る .claude/skills と ~/.claude/skills の配置ルールを確認できます。
