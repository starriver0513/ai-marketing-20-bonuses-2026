BONUS 03 / AI TOOLS COMPLETE GUIDE

導入必須AIツール完全解説

AIツールは、たくさん契約した人が勝つものではありません。自分の仕事に必要な役割だけを選び、情報を入れ、出力を確認し、改善し続ける人が勝ちます。このページでは、推論・リサーチ・音声・文字起こし・画像・動画・API連携まで、初心者が迷わず導入できる順番で整理します。

推論AI音声・文字起こし画像・動画API・自動化低コスト運用
01

はじめに

特典3へようこそ。このページは「AIツール名を覚えるための一覧」ではありません。あなたの仕事をAIで進めるために、どの場面でどのAIを使い、何を入れ、どこを人間が確認するのかを順番に理解するための実践マニュアルです。

02

この特典3でできるようになること

AI初心者がつまずきやすいのは、ツールを増やしすぎることです。まずは役割を分けて、必要なものだけを入れてください。

01 最初に契約するAIを決める

ChatGPT、Claude、Geminiを何となく全部契約するのではなく、自分の作業内容に合わせて優先順位を決めます。

02 AIを役割で使い分ける

考えるAI、調べるAI、整理するAI、録音するツール、画像・動画を作るツールを分けて考えます。

03 安く始める判断軸を持つ

最初はメインAI1つからで十分です。必要になったら補助ツールを追加し、毎月使っていないものは止めます。

04 出力品質を安定させる

AIに丸投げせず、目的、背景、素材、確認基準を渡して、出力を改善するループまで作ります。

03

結論：初心者はこの順番で入れればOK

最初から全ツールを使いこなす必要はありません。まずは以下の順番で導入してください。

01 メインAIを1つ決める

最初はCodex / ChatGPT系、またはClaude系のどちらかをメインにします。導入済みのデスクトップ環境をそのまま使えばOKです。

02 リサーチ用にNotebookLMやGeminiを使う

YouTube、PDF、ドキュメント、記事などを整理し、与えた情報に沿って確認したい時に使います。

03 録音・文字起こしの導線を用意する

Zoom、Loom、Plaud、音声入力ツールを使って、会議・アイデア・講座・質問回答をテキスト化できる状態にします。

04 画像・動画は必要な時だけ足す

サムネ、図解、スライド、ショート動画など、売上や運用に必要な場面から導入します。

05 API連携は最後でOK

Googleスプレッドシート、Chatwork、X、YouTubeなどの自動化は、手動運用で型が固まってから入れます。

04

AIツール選定で失敗する人の共通点

AIが使えないのではなく、使い方の順番がズレているケースがほとんどです。

NG 01 ツール名から選ぶ

「流行っているから契約する」だと、何に使うかが曖昧なまま月額費用だけ増えます。先にタスクを決めます。

NG 02 情報を渡さずに頼む

AIに「SNS投稿作って」「台本書いて」だけでは、一般論になりやすいです。材料を渡して初めて実務レベルになります。

NG 03 1回の出力で判断する

最初の出力が微妙でも、修正内容を返せば精度は上がります。AIは一発勝負ではなく改善前提で使います。

NG 04 費用を管理しない

サブスクとAPI課金は気づかないうちに増えます。月1回、使っているツールと費用を棚卸ししてください。

05

AI活用は「要件定義」から始める

最初に決めるべきことは、どのAIを使うかではありません。「何を任せたいのか」です。YouTube台本なのか、X投稿なのか、講座資料なのか、LINE配信なのか。目的が曖昧なままツールを入れると、AIは便利なおもちゃで終わります。

06

AIプロジェクト作成の5ステップ

この5ステップで考えると、AIツールの使い分けが一気に分かりやすくなります。

01 要件定義

任せたい作業、完成物、対象者、使う場所を決めます。例：YouTube台本を作る、講座生向けQ&Aを作る、X投稿を作る。

02 情報準備

過去台本、商品情報、顧客の悩み、競合の文字起こし、FAQ、実績などを用意します。ここが薄いと出力も薄くなります。

03 プロンプト作成

自分で長文プロンプトを書く必要はありません。目的と背景をAIに伝え、足りない情報を質問させ、プロンプト自体を作らせます。

04 プロジェクト化

ChatGPT、Claude、Gemini、Codex、Claude Codeなどのプロジェクト機能や作業フォルダに、使う情報とルールをまとめます。

05 デバッグ投入

実際に出力させ、ズレた箇所を直し、プロンプトや情報を更新します。ここまでやって初めて実務用AIになります。

07

まず入れるべきAIツール全体マップ

以下のように役割で分けると、ツール選定で迷いにくくなります。

考える ChatGPT / Codex / Claude

文章作成、構成、壁打ち、ファイル編集、実装、レビューなどを担当します。普段の仕事の中心に置くAIです。

調べる Gemini / NotebookLM

長い資料、動画、PDF、Web情報、社内ドキュメントを整理して、理解・要約・比較する場面に向いています。

記録する Loom / Zoom / Plaud / 音声入力

会議、講座、FB、アイデアを録音・録画し、あとでAIに渡せるテキスト資産に変えます。

作る Canva / 画像生成AI / スライドAI / 動画AI

サムネ、バナー、図解、スライド、ショート動画などの制作工程で使います。最終調整は人間が確認します。

つなぐ Google Workspace / Chatwork / X API / GAS

運用が固まった後、投稿、集計、通知、整理を自動化するために使います。最初から触らなくて大丈夫です。

08

ChatGPT / Codexを使う場面

ChatGPTは、文章、壁打ち、構成、調査、画像、GPTs作成まで幅広く使える万能AIです。Codexは、ローカルフォルダを開いて、HTML、Markdown、台本、ナレッジ、設定ファイルを直接扱いたい時に強いです。ローカルフォルダを開いて使うと、単発チャットよりも作業対象を固定しやすくなります。

09

Claude / Claude Codeを使う場面

Claudeは、長文の理解、自然な日本語、文章のトーン調整、講座資料や台本の整理に強い選択肢です。Claude Codeは、ローカルフォルダ内のファイルを読みながら、台本、記事、HTML、設定ファイル、タスク管理まで一気通貫で扱いたい時に使えます。Codexと同じく、フォルダに情報を置いて改善するほど強くなります。

10

Geminiを使う場面

Geminiは、Googleドキュメント、スプレッドシート、YouTube、検索、長い資料との相性が高い場面で使いやすいAIです。大量の情報を読み、比較し、整理し、Google系の作業に寄せていきたい時に候補になります。最新モデルやプランは変わるため、契約前に公式ページで確認してください。

11

NotebookLMを使う場面

NotebookLMは、アップロードした資料、URL、PDF、動画などをもとに内容確認したい時に便利です。特に、講座資料、過去セミナー、Q&A、商品資料、競合リサーチを入れて「この資料の範囲で答えて」と確認する用途に向いています。感情表現やコピーを作るAIというより、情報整理と確認のAIとして使うと強いです。

12

音声入力・録音ツールを入れる理由

AI活用で差がつくのは、プロンプトの知識だけではありません。自分の考え、会議、顧客の悩み、講座の説明、FBをどれだけテキスト化できるかです。テキスト化できれば、AIに整理、要約、教材化、投稿化、Q&A化を任せられます。

13

Loom / Zoom / Plaudの使い分け

録音・録画系は、目的別に分けると迷いません。

Loom 画面付きの解説・FB

画面を見せながら説明したい時に使います。講座の補足、制作物へのFB、操作説明、クライアント共有に向いています。

Zoom セミナー・MTG

長時間のセミナー、Q&A会、打ち合わせを残す時に使います。録画・文字起こしをあとで教材化できます。

Plaud 移動中・対面・思考メモ

机の前にいない時の会話やアイデアを残す用途に向いています。音声を逃さず、あとから要約・ナレッジ化します。

音声入力 3〜10分で背景を一気に渡す

長文を打つのが面倒な時は、音声で背景を話し、AIに整理させます。初心者ほどこの方法が楽です。

14

文字起こしの基本方針

音声や動画をAI活用に使う時は、以下の順で整えます。

01 録る

Zoom、Loom、Plaud、スマホ録音などで、会議・講義・アイデアを残します。

02 文字起こしする

ツール内の文字起こし、Gemini系の文字起こし、音声認識APIなどを使ってテキスト化します。

03 誤字を補正する

固有名詞、商品名、AI用語、人物名は間違いやすいので、AIに補正させ、人間がざっと確認します。

04 用途別に整える

教材用、投稿用、Q&A用、ナレッジ用など、最終用途に合わせて整形します。

05 次回以降に使える形で保存する

その場限りにせず、あとからAIが読める場所へ保存します。保存先は後で迷わないように決めておきます。

15

画像・サムネ・スライド系AIの考え方

画像生成AIやスライドAIは強力ですが、最初からそこに時間を使いすぎると本質からズレます。まずは「何を伝えるか」「誰に刺すか」「どんな導線にするか」を決め、その後でサムネ・図解・スライドに落とし込んでください。画像やスライドは、戦略の代わりではなく、戦略を伝わりやすくするための道具です。

16

動画AI・動画編集AIの考え方

動画領域では、台本、話し方、構成、サムネ、タイトル、導線が先です。自動カットや字幕生成、動画生成AIは便利ですが、伸びない企画を編集で救うことはできません。YouTubeやInstagramでは、媒体ごとの制作手順に合わせて調整します。

17

API連携はいつ始めるべきか

API連携は、最初から触らなくて大丈夫です。まずは手動で「この作業は毎回同じ流れで発生する」と分かるところまで運用します。その後、Googleスプレッドシート、Chatwork、X、YouTube、UTAGEなどのAPIを使って、自動取得・通知・投稿・集計に進みます。APIキーや顧客情報を扱う場合は、必ず権限管理と公開前チェックを入れてください。

18

最安値運用の基本ルール

AIツール費用は、放置すると増えます。最初は小さく始めて、必要なものだけ残してください。

RULE 01 最初はメインAI1つでよい

まずはCodex / ChatGPT系またはClaude系のどちらかを中心にして、毎日使う状態を作ります。

RULE 02 補助AIは用途が出てから追加

リサーチで困ったらNotebookLM、Google連携で困ったらGemini、画面説明が多いならLoom、という順番で足します。

RULE 03 年払いは慎重にする

AIツールは変化が早いため、最初は月額で試し、継続利用が確定してから年払いを検討します。

RULE 04 APIは上限を決める

APIは従量課金です。使い始める時は、用途、上限、ログ、停止条件を決めてから運用します。

RULE 05 月1回棚卸しする

契約中ツール、料金、用途、最終利用日を確認し、使っていないものは止めます。

19

出力クオリティを最大化する5条件

どのAIを使う場合でも、品質を決める条件はほぼ共通です。

01 目的を明確にする

何を作るのか、誰に届けるのか、どんな行動をしてほしいのかを先に決めます。

02 背景を渡す

なぜ作るのか、今どんな状況か、過去に何が起きたのかを渡します。背景が薄いと出力も薄くなります。

03 材料を渡す

実績、商品情報、顧客の声、過去投稿、台本、競合分析など、AIが使える材料を入れます。

04 判断基準を渡す

何を良しとするか、何を避けるか、どんな文体にするかを伝えます。

05 FBを返す

出力に対して「どこが違うか」「なぜ違うか」を返すと、次の出力が良くなります。

ツールを増やす前に確認すること

新しいAIが出てくるたびに契約していると、運用が複雑になります。追加前に以下を確認してください。

CHECK 01 今のAIで本当にできないか

プロンプトや情報の渡し方を改善すれば解決する場合があります。まずは今あるAIで試します。

CHECK 02 月に何回使うか

1回だけ使う用途なら無料枠や単発利用で十分です。毎週使うなら課金候補になります。

CHECK 03 誰が管理するか

アカウント、支払い、APIキー、素材、出力物の管理者が曖昧だと運用が崩れます。

CHECK 04 既存フローに組み込めるか

便利でも、今の仕事の流れに入らないツールは定着しません。使う場面を決めてから契約します。

26

よくある質問

初心者が迷いやすいポイントを先に潰しておきます。

Q1 全部のAIを契約した方がいい？

不要です。最初はメインAI1つで十分です。足りない用途が明確になったら追加してください。

Q2 無料AIだけでできますか？

練習はできます。ただし仕事で使うなら、回数制限、速度、精度、保存性を考えて、最低限の有料プランを検討してください。

Q3 NotebookLMだけで全部できますか？

情報確認には強いですが、コピー制作や感情表現、実装、ファイル編集は別のAIと組み合わせた方が進めやすいです。

Q4 APIは初心者も必要ですか？

最初は不要です。毎回同じ作業を手動で回せるようになってから、自動化候補として考えます。

Q5 どのAIが一番強いですか？

一番強いAIを探すより、自分の作業場に置いて毎日使えるAIを決める方が成果につながります。
