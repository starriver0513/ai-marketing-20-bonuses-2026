BONUS 05 / AI SECURITY SETTINGS

鉄壁セキュリティ対策設定10選

AIでSNS運用、API連携、文字起こし、Google Workspace、UTAGE、GitHub公開を扱う前に、最初に設定するセキュリティ項目をまとめました。対象はAI初心者です。APIキー、顧客情報、講座資料、未公開コンテンツを外部に出さずに、Codex / Claude Code / ChatGPT / Google / GitHubを使うための実務手順です。

01

はじめに：AIを使う前に必ず設定すること

AI活用で問題が起きやすいのは、難しいコードを書いた時ではありません。APIキー、顧客情報、未公開コンテンツ、管理画面URL、共有権限を、よく分からないままAIや公開ページに渡した時です。このページでは、初心者が最初に決めるべき安全ルールを、作業順に整理します。

  • AIに見せる情報と、見せない情報を最初に分けます。
  • APIキーやOAuth tokenは、チャット欄、Google Docs、Chatwork、LINE、公開ページに貼りません。
  • 削除、外部送信、公開、権限変更、決済、会員情報に関わる作業は、実行前に人間確認を入れます。
  • CodexやClaude Codeには、作業してよいフォルダ、触ってはいけない情報、公開前チェックの条件を書いておきます。
  • 不安な操作は、実行前に「この作業で外部に出る情報はありますか？」とAIに確認します。
02

最初に覚えるセキュリティ用語

用語を理解しておくと、AIへの指示が具体的になります。細かい技術理解より先に、何を外部に出してはいけないのかを判断できる状態にします。

API KEY APIキー

外部サービスをプログラムから使うための認証情報です。OpenAI、Anthropic、Groq、X、Googleなどで発行します。第三者に見られると、利用枠や機能を使われる可能性があります。

TOKEN OAuth token

Google Drive、YouTube、Xなどにログイン連携した時に発行される認証情報です。APIキーと同じく、チャットや公開ページに出してはいけません。

ENV .env

APIキーやOAuth tokenをローカルで管理するためのファイルです。AIには値を貼らず、「.envを読んで使ってください」と伝えます。

PERMISSION 権限

閲覧者、編集者、管理者などのアクセス範囲です。最初は閲覧者にし、編集者や管理者は必要な人だけに限定します。

PUBLIC 公開

URLを知っている人、またはインターネット上の人が見られる状態です。公開前に秘密情報、顧客情報、内部情報が混ざっていないか確認します。

LOG ログ

チャット履歴、実行結果、エラー表示、録画、スクリーンショットに残る情報です。秘密情報はログにも残さない運用にします。

03

Claude Codeで必ず入れる6つの防御

AIにローカル作業を任せる場合、APIキーや顧客情報を守るために、最初に防御設定を入れます。特に.env、履歴、実行権限、公開前レビューは必須です。

01 .envファイルを隠す

APIキーやトークンは.envに入れ、AIが.envを読めないようDenyルールを設定します。

02 file-historyを定期削除する

編集履歴に秘密情報が残る可能性があるため、履歴ファイルを定期的に確認・削除します。

03 Sandboxを有効化する

AIが触れる範囲を限定し、不要なフォルダや外部公開物にアクセスさせないようにします。

04 CLAUDE.md / AGENTS.mdに安全ルールを書く

削除、上書き、外部送信、秘密情報の扱い、公開前確認のルールを明文化します。

05 テストしてから公開する

API連携や自動投稿は、テスト環境で確認してから本番へ進めます。

06 /compact → /clearで記憶を切り替える

長い作業で不要な情報が残った場合は、要約してからセッションを切り替え、秘密情報を持ち越さないようにします。

04

鉄壁セキュリティ対策設定10選

まずはこの10項目を設定します。完璧な専門知識よりも、毎回同じ確認を行うことが重要です。

01 全アカウントに2段階認証を設定する

Google、GitHub、ChatGPT、Claude、X、Stripe、UTAGEなど、業務で使うアカウントは2段階認証を有効にします。パスワードは使い回さず、管理ツールで個別に保管します。

02 APIキーをチャット欄に貼らない

Codex、Claude Code、ChatGPT、Chatwork、LINE、Google DocsにAPIキーを直接貼りません。AIにはキーの値ではなく、変数名と保存場所だけを伝えます。

03 .envで秘密情報を管理する

APIキー、OAuth token、Client Secretは.envに保存します。値は自分で入力し、AIには「.envを読んで使ってください」とだけ伝えます。

04 AIの作業フォルダを限定する

デスクトップアプリで作業する時は、必要なプロジェクトフォルダだけを開きます。ホームフォルダ全体、Downloads全体、Drive全体を作業対象にしません。

05 共有権限は閲覧者から始める

Google Docs、Sheets、Drive、GitHub、Notionなどの共有は、最初は閲覧者にします。編集者、管理者、全体公開は、必要性を確認してから設定します。

06 公開前に秘密情報と個人情報を確認する

GitHub Pages、UTAGE、Google Docs公開、SNS投稿の前に、APIキー、OAuth token、顧客名、メールアドレス、電話番号、決済情報、内部URLが含まれていないか確認します。

07 画面共有と録画で秘密情報を見せない

.env、API管理画面、管理者画面、顧客リスト、売上管理画面を開いたまま録画しません。映った場合は、公開前に削除または非表示処理を行います。

08 連携アプリの権限を確認する

文字起こし、SNS自動化、Chrome拡張、外部ツールを使う時は、要求される権限を確認します。必要以上に広い権限を求めるツールは、導入前に代替案を検討します。

09 削除・送信・公開・権限変更は確認後に実行する

AIに作業させる時は、削除、外部送信、公開、権限変更、課金、会員情報変更を勝手に実行させません。対象、実行内容、戻し方を確認してから進めます。

10 流出の疑いがあるキーはすぐ無効化して再発行する

APIキーやtokenがチャット、公開ページ、録画、GitHub、共有ファイルに出た可能性がある場合は、使用を止めて無効化し、新しいキーを発行します。

05

AIに見せてよい情報・見せない情報

AIに渡す情報の判断に迷う場合は、以下の基準で分けます。判断できない時は、値や本文を渡す前に安全確認だけ依頼します。

OK 見せてよい情報

公開済みURL、公開済み投稿、自分で作った原稿、空のテンプレート、変数名、個人情報を削除したサンプルデータ。

CHECK 事前確認が必要な情報

顧客の声、売上資料、社内運用、Chatworkのやり取り、講座メモ、会員限定コンテンツ、未公開の台本。

NG 見せない情報

APIキー、OAuth token、パスワード、2段階認証コード、バックアップコード、顧客のメールアドレス、電話番号、決済情報。

PUBLIC NG 公開しない情報

.env、tokenファイル、実顧客リスト、管理画面URL、会員サイトの内部URL、外部公開していない資料一式。

06

AIに絶対入れない6種類

便利だからといって、AIに何でも貼ってはいけません。以下は対策をしたうえでもチャット本文や公開ナレッジに入れない前提で扱います。

AUTH 認証情報

APIキー、パスワード、トークン、2FAコード、OTPは入れません。必要な場合は.envやSecretsで使わせます。

PAY 決済・銀行情報

カード番号、セキュリティコード、銀行口座番号、暗証番号は扱わせません。

ID 本人確認情報

マイナンバー、パスポート番号、運転免許証番号などはAIに渡しません。

OAUTH Webhook / OAuth秘密情報

WebhookシークレットやOAuth refresh tokenは、漏れると外部連携を乗っ取られる可能性があります。

URL トークン付き限定URL

限定動画、限定マニュアル、決済画面などのトークン付きURLは、実質パスワードとして扱います。

CRYPTO 暗号資産の秘密情報

秘密鍵、シードフレーズ、復元用の文字列は絶対にAIへ渡しません。

07

条件付きで扱える情報の3条件

コンテンツ販売では、顧客情報、売上、Q&A、契約情報を完全に避けると業務が止まります。扱う場合は、匿名化、最小化、適切なツールの3条件を満たします。

MASK 匿名化する

氏名、メール、電話、住所、LINE ID、顧客名は伏せ字化し、本人特定できる情報を外します。

MIN 必要最小限にする

売上やKPIは集計値にし、不要列、不要行、過去データを削ってから渡します。

PLAN 適切なプランで扱う

無料版には公開情報のみ。顧客情報を扱う業務はTeam / Business以上を標準にします。

CHECK 送信前に一度止まる

エラー解決や文章作成でも、貼る前に中身を見ます。スクショにも顧客名やキーが映ることがあります。

08

Codex / Claude Codeで作業フォルダを限定する

デスクトップアプリで使う場合は、最初に開くフォルダを限定します。AIに関係ないフォルダまで見せると、不要な情報を読んだり、誤って編集対象にしたりする可能性があります。

  • 作業対象のプロジェクトフォルダだけを開きます。
  • Downloads、Desktop、ホームフォルダ全体、Google Drive全体を作業対象にしません。
  • AGENTS.mdまたはCLAUDE.mdに、触ってよいフォルダと触ってはいけないフォルダを書きます。
  • 削除や上書きが必要な場合は、対象ファイル名と理由を先に出させます。
  • 顧客情報、決済情報、APIキーを含むフォルダは、必要な時だけ限定的に扱います。
09

AIに触らせる範囲を4つに分ける

AIエージェントはファイルを読み、編集し、コマンドを実行できます。だから「見せる」「編集する」「削除する」「外部送信する」を分けて許可します。

01 読み取り範囲を決める

作業対象フォルダ、素材フォルダ、出力フォルダだけを渡します。顧客名簿、決済、認証情報、素材原本は分けます。

02 編集範囲を決める

編集してよいファイルと、読むだけのファイルを分けます。元本はコピーしてから加工するのが安全です。

03 削除を止める

ファイル削除、DB更新、公開ページ上書きは自動化せず、人間確認を入れます。

04 外部送信を止める

SNS投稿、メール、LINE、Chatwork送信、決済、請求、広告予算変更は承認前に実行しません。

10

公開前チェックの手順

GitHub Pages、UTAGE、Google Docs、SNS投稿など、外部に出るものは公開前に同じ順番で確認します。

01 公開対象を確認する

対象ファイル、対象URL、対象フォルダ、公開先を確認します。何を公開するのか曖昧なまま進めません。

02 秘密情報を確認する

APIキー、OAuth token、パスワード、認証コード、Client Secret、.env、tokenファイルが含まれていないか確認します。

03 顧客情報を確認する

氏名、メールアドレス、電話番号、決済情報、会員ID、個別相談の内容が含まれていないか確認します。

04 内部情報を確認する

管理画面URL、内部チャットURL、未公開の講座資料、配布前のプロンプト、社内用の手順が含まれていないか確認します。

05 戻し方を確認する

公開後に問題が見つかった時、どのファイルを戻すのか、どのURLを非公開にするのかを確認してから公開します。

11

Google Docs / Drive / Sheetsの共有設定

Google Workspaceは便利ですが、共有範囲を間違えると内部情報が外部に出ます。共有前に権限を確認します。

01 最初は非公開で作る

作成直後は自分だけが見られる状態にします。外部共有は内容確認が終わってから行います。

02 共有する時は閲覧者にする

相手に編集してもらう必要がない場合は閲覧者にします。編集者は必要な人だけに限定します。

03 リンク共有の範囲を確認する

「リンクを知っている全員」になっている場合は、公開してよい内容か確認します。社内用や会員限定の資料は範囲を限定します。

04 コメントや履歴も確認する

本文だけでなく、コメント、提案、変更履歴、ファイル名にも内部情報が残る場合があります。共有前に確認します。

05 共有後に権限を見直す

作業が終わったら編集権限を外し、不要な共有相手を削除します。

12

外注・退職者アカウント処置

外注、業務委託、退職者、担当変更が発生した時は、感情や信頼ではなく手順で処理します。アカウントが残ったままになると、顧客情報、講座素材、APIキー、広告アカウント、決済情報へ後からアクセスできてしまいます。

01 対象者の権限一覧を作る

Google Drive、Chatwork、UTAGE、GitHub、SNS、広告、決済、API管理画面を一覧化します。

02 共有ドライブとDocsの権限を外す

個別ファイルだけでなく、親フォルダ、共有ドライブ、リンク共有の設定も確認します。

03 外部サービスのメンバー権限を削除する

管理者、編集者、閲覧者、API利用者、Webhook設定者を分けて確認します。

04 APIキーとトークンを再発行する

本人が見た可能性があるキーは、削除または再発行します。.envだけを消しても外部側のキーが有効なら意味がありません。

05 最終アクセス日を確認する

ログイン履歴、ファイル閲覧履歴、公開設定変更履歴を確認し、異常があれば関係者へ共有します。

13

ログ・アセット監視

セキュリティは設定して終わりではありません。公開物、サムネイル、台本、講座素材、顧客情報、API利用量に異常がないか、定期的に確認します。

LOGIN ログイン履歴

Google、Chatwork、UTAGE、GitHub、SNS、広告アカウントのログイン履歴を確認します。

DRIVE Drive変更履歴

重要フォルダの共有範囲、外部共有、削除、ダウンロード、オーナー変更を確認します。

ASSET 公開アセット

サムネ、LP、HTML、PDF、動画、台本に内部情報や権利上危ない素材が混ざっていないか確認します。

API API利用量

急な利用量増加、エラー増加、知らないアプリ連携がないかを確認します。

SNS SNS投稿履歴

予約投稿、削除済み投稿、DM自動化、外部連携ツールの権限を確認します。

REPORT 月次棚卸し

誰が何にアクセスできるかを月1回棚卸しし、不要な権限を削除します。

14

APIキーをチャットに貼らない運用

AIに設定作業を依頼する時でも、APIキー、アクセストークン、パスワード、顧客情報をチャット本文へ貼らない運用にします。AIには変数名、保存場所、確認手順だけを作らせ、実際の値は本人がローカルの.envへ入力します。

  • AIには.env.exampleを作らせ、.envの実値は本人が入力します。
  • Slack、Chatwork、Google Docs、公開HTMLに認証情報を書きません。
  • 外注者へ渡す場合は、本番キーではなく用途を限定したキーや権限を使います。
  • 退職・契約終了時は、アカウント削除だけでなくAPIキーの再発行も確認します。
  • ログにキーが出ていないか、必要に応じて確認します。
15

広い権限を要求する外部アプリの扱い

便利そうな拡張機能や外部アプリでも、権限が広すぎる場合は慎重に扱います。特にSNS、Google Drive、顧客管理、決済、会員サイトに接続するアプリは、導入前に権限と削除手順を確認します。

権限 何を読めるか確認する

投稿、DM、顧客リスト、Drive内ファイル、決済情報など、どの範囲にアクセスするかを見ます。

期間 使う期間を決める

一度だけ使うツールは、作業後に連携解除します。常時連携が必要なものだけ残します。

代替 必要ならAPI/CLI方式へ寄せる

広い画面操作権限を要求するアプリより、用途を限定したAPIやCLIで足りる場合があります。

解除 削除手順を残す

導入時に、どこから連携解除するか、誰が解除するか、解除後に何を確認するかを記録します。

16

UTAGE / Chatwork / 顧客情報の扱い方

マーケティング実務では、API連携やチャット履歴に顧客情報が含まれます。公開コンテンツに混ざらないよう、扱う範囲を分けます。

UTAGE 会員サイト・ファネル・決済は確認必須

ページ、シナリオ、決済、会員情報に関わる変更は、対象ID、変更内容、戻し方を確認してから実行します。APIキーの値は出しません。

CHATWORK チャット全文をそのまま公開しない

公開用には、個人名、ID、内部URL、顧客情報、未公開内容を削除してから使います。必要な場合は要点だけに整理します。

CUSTOMER 顧客情報は匿名化する

氏名、メールアドレス、電話番号、購入履歴、相談内容は、公開前に削除または匿名化します。

SALES 売上・決済情報は集計値で扱う

個別の購入者情報や決済画面を出さず、必要な場合は集計値や公開してよい実績だけを使います。

17

画面共有・Loom撮影チェック

画面共有や録画では、意図せず秘密情報が映ることがあります。録画開始前と公開前に確認します。

BEFORE 録画前に閉じるもの

.env、API管理画面、Google Cloud、OpenAI Platform、Anthropic Console、UTAGE管理画面、決済管理画面、顧客リスト。

DURING 操作中に注意するもの

ブラウザのタブ名、URL、通知、ダウンロードフォルダ、チャット履歴、ファイル名、ターミナルの実行結果。

AFTER 公開前に確認するもの

APIキー、顧客情報、内部URL、管理画面、会員限定情報が映っていないか確認します。映った場合は削除または非表示処理をします。

IF SHOWN 映った場合の対応

APIキーやtokenが映った可能性がある場合は、該当キーを無効化し、新しいキーを発行します。

18

スクショ・画面共有・録画で漏れやすい場所

セキュリティ事故は、本文だけではなく画面にも起きます。共有前、録画前、公開前に以下を確認します。

URL URL欄・タブ・ブックマーク

限定URL、顧客名、案件名、管理画面タイトルが映っていないか確認します。

NOTICE 通知ポップアップ

Slack、Chatwork、LINE、メール通知に顧客名、金額、相談内容が出ないよう集中モードにします。

TERM ターミナル履歴

環境変数、キー、過去コマンドが表示されないよう、画面共有前に整理します。

VIDEO 動画の全フレーム確認

録画は1フレームでも漏れればスクショ化されます。公開前に等倍で確認し、必要箇所はぼかします。

19

OAuth / 連携アプリの確認

外部アプリやChrome拡張を使う場合は、何にアクセスできる権限を求めているかを確認します。便利そうに見えても、権限が広すぎる場合は導入前に止まります。

  • Google Drive全体、Gmail全体、画面全体、全ファイルへのアクセスを求める場合は、目的に対して必要か確認します。
  • 文字起こしやSNS自動化は、公式APIや必要最小限の権限で実行できる方法を優先します。
  • 使わなくなった連携アプリは、Google、GitHub、X、Notionなどの設定画面から解除します。
  • OAuth tokenはAPIキーと同じ扱いです。チャット、公開ページ、録画に出しません。
  • 権限が分からない時は、AIに「この権限で何ができるか」を説明させてから判断します。
20

APIキー再発行の判断基準

APIキーは、何も問題がないのに頻繁に作り直す必要はありません。ただし、外部に出た可能性がある時はすぐに対応します。

通常時 同じキーを使い続けてOK

安全に.envで管理できており、公開や共有に含まれていない場合は、無理に再発行する必要はありません。

再発行 外部に出た可能性がある時

チャット、公開ページ、GitHub、録画、スクリーンショット、共有ドキュメントに出た可能性があれば、無効化して再発行します。

更新 .envと連携設定を更新する

新しいキーを発行したら、.env、サービスの設定画面、自動化スクリプト、デプロイ環境の環境変数を更新します。

確認 古いキーが使えないことを確認する

古いキーを無効化し、不要なtokenや連携アプリも削除します。

21

漏洩疑い時は削除ではなくローテーションする

APIキーやトークンは、チャットやGitHubから消しただけでは安全になりません。第三者がすでに見た前提で、該当キーを失効し、新しいキーに差し替える必要があります。

  • キーが画面共有、スクショ、録画、ログ、GitHub、Chatworkに出た可能性がある場合は即ローテーションします。
  • 古いキーを無効化し、新しいキーを発行し、.envやSecretsだけを更新します。
  • 漏れた場所を削除しても、コピー・検索キャッシュ・通知ログに残る可能性があります。
  • 復旧後は、どこに出たか、誰が見たか、再発防止の設定をログに残します。
22

SNSや講座外で話してよい範囲

講座や実務で学んだ内容を発信する時は、出してよい範囲を分けます。特に手順、テンプレート、内部資料、配布物は注意します。

OK 話してよい内容

何を学んだか、なぜ重要か、自分が取り組む内容、公開済みツール名、一般的な気づき。

NG そのまま出さない内容

講座内の手順書、テンプレート全文、配布プロンプト、内部チャット、会員限定ページ、未公開の制作物。

CHECK 迷う内容

具体手順、設定値、スクリーンショット、ツール連携の詳細、顧客事例、売上資料。外部に出す前に確認します。

SAFE 安全な出し方

個別名や内部情報を削除し、要点だけを自分の言葉で説明します。手順やテンプレートは会員向けの中に残します。

最後のチェックリスト

AI活用を始める前、公開前、外部連携前にこの項目を確認します。1つでも不安があれば、作業を止めて確認します。

  • 主要アカウントに2段階認証を設定した。
  • APIキーやtokenをチャット欄に貼っていない。
  • .envに値を保存し、AIには変数名だけ伝えている。
  • Codex / Claude Codeの作業フォルダを限定している。
  • 削除、外部送信、公開、権限変更は確認後に実行するルールにしている。
  • 公開前にAPIキー、顧客情報、内部URL、未公開資料を確認している。
  • Google DriveやDocsの共有権限を閲覧者から設定している。
  • 画面共有や録画に秘密情報が映っていない。
  • 外部に出た可能性があるキーは無効化し、再発行する判断ができる。
  • 迷う情報は、値を出す前に安全確認だけAIに依頼する。

外部公開前のセキュリティ停止プロトコル

秘密情報が混入した可能性がある時は、作業を続行せずに止めます。公開前に止まれる設計が、炎上と情報漏えいを防ぎます。

01 まず公開を止める

GitHub Pages、UTAGE、SNS投稿、予約投稿を止め、外部公開を一時停止します。

02 秘密情報を探す

APIキー、トークン、Webhook URL、顧客情報、社内URL、個人情報が含まれていないか検索します。

03 漏れた可能性があるキーを再発行する

公開済みHTML、Git履歴、チャットログ、Slack本文に出た可能性があるキーは、削除だけでなく再発行します。

04 権限を棚卸しする

外注終了者、退職者、不要な共有アカウントのアクセス権限を剥奪します。

05 再発防止ルールを追記する

CLAUDE.md / AGENTS.md、チェックリスト、Skillの品質チェックへ、今回の防止ルールを追加します。
