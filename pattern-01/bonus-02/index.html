BONUS 02 / LOCAL AI DIRECTORY

THE・DIRECTORY自分最適のAI経営フォルダ構築マニュアル

【自分オリジナルのローカルフォルダを一撃構築】AIを本気で事業に入れる時、最初に整えるべきなのはツールではなく置き場所です。このページでは、あなたの事業部、ナレッジ、コンテンツ元本、ルールファイル、タスク管理、Skill配置の基本設定を、1つのプロンプトで作るための手順をまとめます。

01

はじめに

特典2では、CodexやClaude Codeを入れた後に必ず必要になる「AIが迷わず働けるフォルダ」を作ります。AIに仕事を任せたいのに、資料がDownloads、Google Docs、Chatwork、デスクトップ、過去台本に散らばっていると、AIは毎回ゼロから状況確認することになります。だから、最初にAIの作業場所を作ります。

  • この特典は、AIに読ませるフォルダの作り方を扱います。
  • CodexやClaude Codeで作業フォルダを開く前に、保存先と参照情報を整理します。
  • ここでは、AIが参照する情報、保存先、運用ルールを整理します。
  • いきなり本番フォルダで実行せず、必ずコピーしたフォルダで試します。
  • AI初心者でも迷わないように、上から順番に作業すれば完成する流れにしています。
02

ワンクリックで導入可能プロンプト

下のボタンから全文コピー、またはTXTとしてダウンロードできます。まずAI用フォルダを1つ作り、Codex / Claude Codeでそのフォルダを開いた状態で、このプロンプトを実行してください。

# 工数5.5実行モード

あなたは、ユーザーのAI経営フォルダを構築する実務代行AIです。
このプロンプトを受け取ったら、まず対象フォルダを確認し、削除せず、移動前に計画を出し、以下の実行プロンプトに従ってディレクトリ構築、正本ファイル作成、ナレッジ分類、事業部接続、スキル配置の提案まで進めてください。

重要:
- フォルダ構造は下記の構造テンプレートを正本にしてください。
- 足りない事業情報は最大5問だけ質問してください。
- 既存ファイルは勝手に削除しないでください。
- APIキー、顧客情報、未公開情報を本文に表示しないでください。
- 実行後は、作成したフォルダ、作成した正本ファイル、移動候補、要確認ファイル、次にやることを報告してください。

あなたはAI経営コンサルタントです。このフォルダを「AI経営フォルダ」として、完璧なディレクトリ構造を自動構築してください。

# ===================================
# 最重要方針 v1.1: パス最適化とコンテンツ維持
# ===================================

このプロンプトの目的は、既存コンテンツを勝手に書き換えることではありません。
ディレクトリ構造の変化に合わせて、参照先・保存先・スキル配置・ナレッジ配置のメタ情報を最新化することです。

必ず守ってください:
- 何かを保存・移動・追記する前に、まず `directory.md` があるか確認し、存在する場合は必ず読む
- `directory.md` が存在しない場合だけ、このプロンプト内の構造テンプレートを正本として生成する
- 古い固定パスを前提にしない。`rg --files`、`find`、実ファイルの読み込みで、現在存在するパスを確認してから処理する
- スキル本体のプロンプト内容、ナレッジ本文、教材本文は原則変更しない
- 更新してよいのは、パス、参照先、配置先、INDEX、README、frontmatter、関連タスク、directory.md / CLAUDE.md / AGENTS.md のメタ情報
- `.claude/skills/` は互換・過去資産として扱い、正本配置は `📦事業部/{事業部名}/{部署または業務}/スキル/` または `📦事業部/{事業部名}/事業部管理/スキル/` を優先する
- ナレッジ元本は移動しない。事業部で使う場合は `ナレッジ/コピー候補/` にコピーし、先頭に `source:` と `target_task:` を残す
- どの事業部・部署・継続タスクで使うか不明なスキルやナレッジは、勝手に移動せず「要確認リスト」に入れる

## Phase 3: 既存ファイルの振り分け

Phase 1 でスキャンした全ファイルを、以下の判断フローで振り分けてください。
すでに `directory.md` が存在する場合は、この判断フローより `directory.md` を優先し、必要に応じてこのv1.1方針を追記してください。

### 振り分け判断フロー

```
ファイルを1つ取り出す
    ↓
Q1. ナレッジテンプレートの1つか?（パーソナル/プロダクト/コンテンツ）
    YES → 🌐共通ナレッジ/📌共通ナレッジ/{該当ナレッジ}/ に配置
    NO  → Q2
    ↓
Q2. 事業部横断で再利用される「抽象的な知識」か?
    YES → 🌐共通ナレッジ/📌共通ナレッジ/{該当カテゴリ}/ に配置
    NO  → Q3
    ↓
Q3. 外部ツールからの未加工データか?（Googleドキュメント/スプレッドシート/音声/動画 等）
    YES → 🌐共通ナレッジ/📌コンテンツ元本/{ツール名}/ に配置
    NO  → Q4
    ↓
Q4. AIスキルまたはスキル関連ファイルか?
    ├─ YES → Q4-A
    └─ NO  → Q5
    ↓
Q4-A. どの事業部・部署・継続タスクで使うか?
    ├─ 明確 → 📦事業部/{事業部名}/{部署または業務}/スキル/{スキル名}/ に配置または参照接続
    ├─ 事業部横断 → 📦事業部/{事業部名}/事業部管理/スキル/{スキル名}/ に配置または参照接続
    └─ 不明 → 要確認リストへ（勝手に中身を書き換えない）
    ↓
Q5. どの事業部の仕事に直結するか?
    ├─ 特定の事業部に属する → 📦事業部/{事業部名}/{部署または業務}/{適切なサブフォルダ}/
    ├─ 複数の事業部で使う → Q2 に戻る（共通ナレッジの可能性）
    └─ どこにも属さない → Q6
    ↓
Q6. 将来使う可能性はあるか?
    YES → 🗂️アーカイブ/ に退避
    NO  → そのまま放置（後で判断）
```

### 振り分けルール

- **元本は1箇所**（同じ元本を複数箇所にばら撒かない）
- 事業部で使うナレッジは、元本を動かさず `ナレッジ/コピー候補/` へコピーし、`source:` と `target_task:` を先頭に残す
- スキル本文は原則変更しない。変更してよいのは、パス、参照先、README、INDEX、frontmatter、関連タスクなどのメタ情報
- `.claude/skills/` から事業部配下へ移す場合は、まずコピーまたはミラーで動作確認し、必要なら互換用READMEやリンクを残す
- ファイル名は `YYYY-MM-DD_{種類}_{テーマ}.md` に統一
- バイナリファイル（音声/動画/画像）は `.md` のメタファイルを隣に作る
- 振り分けに迷ったら事業部側に置く（後で共通ナレッジに昇格できる）

振り分け完了後、以下を報告してください:
- 各フォルダに何ファイル入ったか一覧
- スキル配置の更新件数（.claude配下 / 事業部配下 / 要確認）
- ナレッジコピー候補の作成件数（source / target_task つき）
- 振り分けに迷ったファイルのリスト（あれば）

なぜディレクトリ構造が重要なのか

AIの出力が弱くなる原因は、AIの性能不足だけではありません。むしろ、AIが見るべき資料、守るべきルール、保存すべき場所が曖昧なまま作業していることが大きな原因です。ローカルフォルダは、AIにとっての職場です。机、棚、資料室、担当部署、ルールブックがあるから、AIは同じ品質で働きやすくなります。

  • 資料が散らばると、AIは毎回前提を取り違えやすくなります。
  • 出力先が曖昧だと、完成物、下書き、元本、ナレッジが混ざります。
  • ルールファイルがないと、削除前確認、公開前チェック、口調、保存先が毎回ブレます。
  • 事業部で分けると、SNS、商品、セールス、経理などを別々の担当AIに任せやすくなります。
  • 今後Skillやエージェントチームを作る時も、置き場所が整理されているほど拡張しやすくなります。

よくある失敗と対策

特典2で一番大事なのは、勢いで本番フォルダを壊さないことと、分類を細かくしすぎて使えない構造にしないことです。

MISS 01 本番フォルダでいきなり実行する

必ずコピーで試します。うまくいった構造だけ本番に反映します。

MISS 02 分類を細かくしすぎる

最初は事業部と主要媒体だけで十分です。あとから増やせます。

MISS 03 元本とナレッジを混ぜる

元の録音・文字起こしと、分析済みナレッジは分けます。

MISS 04 directory.mdを更新しない

フォルダだけ増やして保存先一覧を更新しないと、AIが保存先を間違えます。

MISS 05 タスクが散らばる

各事業部のタスク.mdに集約します。媒体ごとにタスクファイルを増やしすぎません。

MISS 06 秘密情報を混ぜる

APIキー、顧客情報、管理画面情報は、公開物や配布用ナレッジから分けます。
